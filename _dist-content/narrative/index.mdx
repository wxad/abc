在工作的这 10 年间，我审视过许多次自己所从事的事业、所追求的目标。去年在 《[做 UI 的开发](https://wxad.design/abc/deng)》 中，我说了许多看起来很丧，而实际上只是希望自己坚定下去的话：

> 我并不认为做 UI 这件事就能够代表什么。我会停止讨论价值。
>
> 渴望燃烧，就是渴望化为灰烬。
>
> 不希求他人，不粉饰自己。滑进瞳孔的一盏盏路灯不是星星，热带的太阳鸟也不会落在我的树上。我只需面向暖流走向海。难得了，足够了。
>
> 我知道顾城说的执者失之。你想当一个诗人的时候，你就失去了诗。你想成为一个人的时候，你就失去了你自己。在你什么也不想要的时候，或许一切才如期而来。

这一大段文字所表达的意思都是一样的。我在等待一个答案，即便不知道问题是什么。我鼓励自己不后悔，即便不知道自己真的错过了什么。在今年拟定工作目标时，我希望自己能够持有更积极的态度。

## 我需要一个叙事

虽然我今年年初就建了一个文档，把所有关于职业和工作的思考记录下来，零零碎碎的，一有感想就写，总共已经接近一万字。但越是记录，显得越是凌乱。如何以一个“连贯的叙事”来陈述自己，仍然是一个问题。关键是，我又不希求于他人的帮助和理解。我认为他人的话帮助不了回答我自己。

我并不认为每个人都需要所谓的叙事，只是当我回望过往时，我无法解答许多事情：

1. 在我过 3-2 的时候，我只陈述了一个关于 [设计语言](https://wxad.design/abc/adui) 的项目，我不提及自己在体验上的任何追求。因为我害怕评委不买账。但是，既然能做出令人满意的作品，我一定具有某种才能，究竟要通过什么方式阐述呢？
2. 我所擅长的交互动画，整体技术难度也不是特别的高；在通用工具上也无法做到很大的影响力。但这是我的问题，还是业务的特点？如果这是业务特点使然，我为什么不讲适配业务特点的故事呢？
3. 不同部门的 UI 开发在做不同的事，AI、SEO、大前端、图形类...即便是我们一个设计中心内不同组的 UI 开发，所专注的事情确实不一样。不一样的是什么？我们的标准还一样吗？

有什么不一样？

## 应用型开发与工具型开发

我将 UI 开发大体两个方向：工具型的开发和应用型的开发。

工具型解决通用问题，讲究效率、聚焦流程；

应用型解决专业问题，讲究效果、聚焦用户。
